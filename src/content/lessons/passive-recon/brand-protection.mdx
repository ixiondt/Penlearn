## Defending the name, not the network

Most recon points outward at a target. Brand protection points the same
techniques *inward* — at attackers impersonating **you**. Before a phishing
campaign lands in inboxes, the attacker registers a lookalike domain, stands up
infrastructure, and often fronts it behind Cloudflare to hide the origin.
Catching that infrastructure early — ideally before the emails go out — is a
defensive win.

`phish-infra-check.sh` enumerates infrastructure impersonating your domain. It's
passive (queries public records and third parties, never the attacker's host
beyond a normal resolution), so it's safe to run on a schedule.

## Running a brand sweep

```bash
./scripts/phish-infra-check.sh acme.com --ct-days 30
```

It chains several lookalike-discovery techniques:

- **dnstwist permutations** — typosquats and homoglyphs (acme.com → acrne.com,
  acme-login.com, аcme.com with a Cyrillic а). Falls back to a pure-bash
  generator if dnstwist isn't installed
- **crt.sh CT-log search** — newly-issued certs for names resembling yours
  (attackers get TLS certs too, and CT logs them)
- **DNS resolution** — which permutations actually resolve (live vs. parked)
- **Cloudflare-fronting check** — NS-pattern match; a lookalike fronted by
  Cloudflare is a strong **evilginx/AiTM** fingerprint (attackers hide origin IPs
  behind the CDN)
- **Shodan favicon hash** (mmh3) — finds hosts serving *your* favicon on *not
  your* infrastructure (cloned login pages)
- **urlscan.io history** — prior scans of the lookalike

## Reading the result

The script exits **2 if live impersonation is detected** — which makes it a clean
signal for automation:

```text
phish-infra-check.sh (nightly cron)
        │  exit 2 = live lookalike found
        ▼
operator review ─▶ osint-passive.sh --ioc <candidate>   (enrich)
        │
        ▼
confirmed malicious ─▶ containment.sh block-ioc / takedown request
```

A lookalike that (a) resolves, (b) serves your favicon, and (c) is Cloudflare-
fronted is almost never coincidence — that's a staged credential-harvesting site.

## Why favicon hashing works

A phishing kit clones your login page wholesale, favicon included. The favicon's
mmh3 hash is a fingerprint; searching Shodan for that hash surfaces every host
serving it. Legitimate matches are your own infrastructure — anything else on an
unfamiliar IP is a clone worth investigating.
