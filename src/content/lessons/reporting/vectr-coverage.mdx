## From "we found things" to "can they detect things"

A pentest report tells the client what's exploitable. A **purple-team** report
tells them what their detection stack actually *caught* while you did it. VECTR
([vectr.io](https://vectr.io)) is the tracking layer for that: it records which
ATT&CK techniques were exercised and whether each was detected, partially
detected, or missed.

`vectr-sync.sh` is the bridge — it pushes your findings into VECTR as test cases
and pulls back the coverage scoring so it can land in the final report.

## Configure once

VECTR connection details come from the environment (or `~/.vectr.env`):

```bash
export VECTR_URL="https://vectr.example.com"
export VECTR_API_KEY="..."
export VECTR_DB="my-assessment-db"

./scripts/vectr-sync.sh status      # confirm reachability
```

`env-check.sh` reports VECTR reachability in its integration section, so you can
confirm wiring before an engagement.

## Push findings (dry-run first)

Always preview what will be created before writing to the VECTR instance:

```bash
# Dry run — shows what would be pushed, changes nothing
./scripts/vectr-sync.sh push-findings --dry-run \
  --findings scored.json --assessment "Client Q1" --campaign "Pentest"

# Live
./scripts/vectr-sync.sh push-findings \
  --findings scored.json --assessment "Client Q1" --campaign "Pentest"
```

Each finding becomes a test case tagged with its ATT&CK technique. The blue team
then marks each one Detected / Partial / Not Detected in VECTR as they review
their telemetry.

## Pull coverage back into the report

Once the blue team has scored the campaign, pull the results:

```bash
./scripts/vectr-sync.sh pull-coverage --campaign-ids "camp-001" --output vectr-coverage.json
```

Then feed that file to the report generator. It adds a **Detection Coverage**
section: a summary table (detected / partial / missed), a coverage-percentage
callout, the per-technique results, and a **Coverage Gaps** subsection listing
every technique no control caught — your detection-engineering backlog.

```bash
python3 scripts/report-generator.py --findings scored.json --output report.docx \
  --vectr-coverage vectr-coverage.json
```

## The purple-team loop

```text
red: run technique ─▶ vectr-sync push-findings ─▶ blue: score in VECTR
                                                        │
report (with coverage + gaps) ◀── vectr-sync pull-coverage ◀──┘
        │
        ▼
   detection engineering closes the gaps ─▶ re-test next cycle
```

The value isn't a one-time number — it's the loop. Each cycle, the gap list
should shrink. A coverage report with no follow-up re-test is just a snapshot;
the point is the trend.
