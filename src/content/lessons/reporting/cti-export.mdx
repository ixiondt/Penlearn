## Sharing what you learned

An incident produces IOCs — IPs, domains, hashes, file paths, accounts. Kept in a
text file they help only you. Exported as **STIX 2.1**, they become something a
TIP (Threat Intelligence Platform), an ISAC, or a partner SOC can ingest
automatically and use to defend themselves. CTI export is how a single incident
turns into community-wide protection.

`cti-export.sh` converts an IOC list into a STIX 2.1 JSON bundle — the standard
structured format for cyber-threat intelligence.

## From IOC list to STIX bundle

Start with the IOCs you collected during the incident (`03-iocs.txt` in the
incident workspace):

```bash
./scripts/cti-export.sh --ioc-file iocs.txt --incident-id INC-2026-001
```

The script classifies each IOC by type (the same IP/domain/hash/path/account
regex used across `sigma-rule-builder.sh` and `pcap-analysis.sh`), wraps each as
a STIX **Indicator** object with a proper pattern, and bundles them with stable
UUIDs.

## Adding attribution context

If the incident maps to known malware or a campaign, attach it — STIX models the
relationships, so consumers see not just the indicators but what they belong to:

```bash
./scripts/cti-export.sh --ioc-file iocs.txt \
  --malware "BlackCat" --campaign "OP-NIGHTFALL" --incident-id INC-2026-001
```

This emits Malware / Campaign STIX Domain Objects and `indicates` relationships
linking the indicators to them — so a downstream analyst can pivot from one IOC
to the whole cluster.

## Where it fits the IR flow

```text
forensics-collect ─▶ pcap-analysis / ioc-enrich ─▶ 03-iocs.txt
                                                        │
                                  ┌─────────────────────┼─────────────────────┐
                                  ▼                      ▼                     ▼
                          sigma-rule-builder      cti-export.sh         timeline-merge
                          (detect locally)     (share as STIX)        (reconstruct)
```

The same IOC file fans out: build local detections, share intelligence, and
reconstruct the timeline. CTI export is the *outbound* arm — turning your
incident into a contribution.

## Hygiene before you share

STIX bundles travel outside your organization, so sanitize first:

- Strip internal-only IOCs (RFC1918 IPs, internal hostnames) unless the recipient
  is trusted and needs them
- Confirm IOCs are actually malicious, not benign infrastructure caught in the net
  — false indicators erode trust in shared intel
- Set appropriate TLP (Traffic Light Protocol) marking with the recipient in mind
